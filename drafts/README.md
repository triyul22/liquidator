# drafts/

Черновики статей до публикации. Каждая статья - своя подпапка `{slug}/`.

## Структура одной статьи

```
drafts/{slug}/
├── brief.json        # агент 1 (семантика)
├── research.json     # агент 2 (юр-ресёрч)
├── outline.json      # агент 3 (архитектор)
├── draft.md          # агент 4 (писатель)
├── uniqueness.json   # агент 5 (антишаблонность)
├── article.html      # агент 6 (SEO-редактор)
├── meta.json         # агент 6 (SEO-редактор)
└── _ready/           # агент 7 в режиме draft
    ├── article.html
    ├── meta.json
    └── assets/
```

## Режимы работы

- **draft** (первые 2 недели) - агент 7 складывает готовое сюда, на сайт ничего не выкатывается. Ручной ревью.
- **live** - агент 7 копирует в `articles/{cat}/`, обновляет `articles.json`, `sitemap.xml`, делает git push.

Переключение: `PUBLISH_MODE=live` в `.env` (по умолчанию `draft`).

## _review/

Если статья 3 раза подряд не прошла антишаблонность - агент 5 кладёт её сюда для ручного разбора.

## _review_queue.json

Список статей в режиме draft, ожидающих ревью.
