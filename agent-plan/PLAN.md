# SEO-агент для сайта "Ликвидатор" (банкротство)

## Цель клиента
Автоматизация: агент пишет 10+ статей в день, уникальность >90%, сам ищет темы, сам публикует на сайт. Клиенту важен только результат - технические детали его не интересуют.

## Решение: Python + статический сайт (без n8n, без WordPress)
Клиент управляет через Telegram-бот. Вся логика на Python, сайт остаётся статическим.

```
├── sitemap.xml         # обновляется при публикации
└── agent/              # Python-агент (не деплоится на сайт)
```

## Пайплайн агента (Python + Claude API)
1. Поиск тем: Google Suggest, Wordstat, частотность 50-500 (низкочастотка для нового сайта)
2. Агент-ресерчер (Claude + web_search): факты, 127-ФЗ, судебная практика 2025-2026
3. Агент-структуролог: план H1-H2-H3 под интент, 2500-4000 слов
4. Агент-писатель: Sonnet 4.6, FAQ-блок для schema.org, внутренние ссылки из articles.json
5. Гуманизация: скилл humanise-text, убрать AI-штампы и длинные тире
6. Уникальность: text.ru или content-watch API, если <90% - переписать слабые абзацы
7. Сохранение: HTML из шаблона, запись в articles.json, обновление sitemap.xml
8. Git commit + push → автодеплой

## Управление для клиента: Telegram-бот
- /статус, /последние, /пауза, /старт
- /добавить тема: ... - ручной приоритет
- Уведомления о каждой публикации

## YMYL требования (критично для банкротства)
- Блок "Об авторе" с реальным юристом в каждой статье
- Дисклеймер "не является юридической консультацией"
- Ссылки на КонсультантПлюс (первоисточники)
- Без этого Google не будет ранжировать

## Инфраструктура
- VPS (Timeweb/Beget) ~500₽/мес

- Cron каждые 2 часа публикует 1 статью (равномерно 10/день)
- IndexNow пинг для Яндекс+Google при публикации
