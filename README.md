# ЛИКВИДАТОР - сайт + SEO-конвейер

Юридическая компания: банкротство физических и юридических лиц, взыскание задолженности.

## Что в репозитории

- `index.html`, `assets/`, `articles/` - публичная часть сайта (то, что видят клиенты)
- `main.py` - сервер на FastAPI: форма заявок + превью-роут для черновиков
- `.claude/agents/` - **системные промпты 7 ИИ-агентов** (заказчик правит здесь)
- `drafts/` - **черновики статей**, ещё не опубликованные (заказчик правит здесь)
- `tools/` - служебные скрипты (эмбеддинги, автоподтяжка из GitHub)

## Инструкция для заказчика

### 1. Где смотреть готовые черновики статей

Открой `https://liquidator.ru/preview/` (логин/пароль выдадим отдельно).
Это закрытая страница - её не видят ни клиенты, ни поисковики.

На каждой статье в правом верхнем углу две кнопки:

- **«✏️ Править текст на GitHub»** - открывает редактор GitHub прямо на этом файле. Можно править абзацы, заголовки, FAQ. После «Commit changes» правки появятся на сервере в течение 5 минут.
- **«⚙️ Правки meta»** - открывает `meta.json` (заголовок, описание, ключевые слова). Правь только то, что понимаешь: `title`, `description`, `h1`. Остальное оставь как есть.

### 2. Где править системные промпты агентов

7 ИИ-агентов лежат в `.claude/agents/` :

1. `1-semantics.md` - подбирает ключевые слова и брифует тему
2. `2-legal-research.md` - собирает юридические факты со ссылками на закон
3. `3-architect.md` - планирует структуру статьи
4. `4-writer.md` - пишет текст
5. `5-factchecker.md` - проверяет факты против research
6. `6-seo-editor.md` - вёрстка HTML, Schema.org, мета-теги
7. `7-publisher.md` - публикация на сайт

Открой нужный файл на GitHub, нажми «✏️», поправь промпт, «Commit changes». Через 5 минут изменения подхватятся на сервере и применятся в следующих запусках конвейера.

**Что обычно правят:**
- Тон и стиль («пиши проще», «убери канцелярит»)
- Длина статьи (`length: 8000-15000`)
- Дисклеймеры и обязательные фразы
- Запрещённые формулировки

### 3. Куда писать комментарии и задачи

Открой [Issues](../../issues/new) на GitHub - это электронный список задач.

Что-то не нравится в статье - заведи issue: «В статье X переделай раздел про МФЦ, факты устарели».
Хочешь поменять подход у агента - issue: «Агент №4 пишет слишком сухо, нужен живой стиль».

Мы видим все issues и берём в работу.

## Для разработчиков (как развернуть)

### Локально

```bash
pip install -r requirements.txt
cp .env.example .env  # отредактировать SMTP, PREVIEW_USER, PREVIEW_PASSWORD
python -m uvicorn main:app --host 127.0.0.1 --port 8765
```

### На сервере (Timeweb Cloud Apps)

1. Подключить репо к Cloud Apps, ветка `main`.
2. ENV переменные: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `MAIL_TO`, `MAIL_FROM`, `MAIL_FROM_NAME`, `PREVIEW_USER`, `PREVIEW_PASSWORD`, `GITHUB_REPO=triyul22/liquidator`, `GITHUB_BRANCH=main`.
3. Команда запуска: `uvicorn main:app --host 0.0.0.0 --port $PORT`.
4. Cron на сервере (для автоподтяжки правок заказчика):
   ```cron
   */5 * * * * cd /app && /usr/bin/python3 tools/git_autopull.py >> /tmp/autopull.log 2>&1
   ```

### Конвейер агентов (запуск только локально, не на сервере)

```bash
# В Claude Code:
/run-pipeline "тема статьи"
```

Агенты сохраняют результат в `drafts/{slug}/`, далее push в GitHub - и заказчик видит черновик в `/preview/`.

## Безопасность

- `.env` НЕ коммитится (есть в `.gitignore`)
- `data/` (SQLite с эмбеддингами) НЕ коммитится
- Превью закрыт Basic Auth + `noindex` (статьи в превью не попадают в Яндекс/Google)
- Прямой доступ к `/drafts/`, `/.claude/`, `/data/`, `/tools/` через сайт заблокирован
